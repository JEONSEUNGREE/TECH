ch11-12~14 LinkedList

일반 배열
장점 : 배열은 구조가 간단하고 데이터를 읽는데 걸리는 시간이 짧다 (access time)

단점1 : 크기를 변경할 수 없다.
-크기를 변경할 수 없다.
-배열 변경순서 1.더큰 배열 생성 2. 배열 복사 3. 참조변경
단점2 : 비순차적인 데이터의 추가, 삭제에 시간이 많이 걸린다.

LinkedList - 배열의 단점 보완
배열과 달리 불연속적으로 존재하는 데이터 연결
데이터 삭제: 단 한 번의 참조변경만으로 가능
데이터 추가: 한번의 Node 객체생성과 두 번의 참조변경만으로 가능

단점 : 연결리스트. 데이터 접근성이 나쁨 (자신의 다음 리스트밖에 모름)
하나 추가할때마다 객체가 하나씩 생성되서 주소값 연결

정리 :
ArrayList 일기가 빠르고 추가/삭제는 느리다  단 순차적인 추가삭제는 더빠름
LinkList 읽기가 느리고 추가/삭제가 빠르다. 데이터가 많을수로 접근성이 떨어짐
배열의 경우 중간에 삭제하면 연속적이기때문에 다 한칸씩 이동.
