 ch7-45~50 내부클래스의 제어자와 접근성

내부 클래스의 제어자는 변수에 사용 가능한 제어자와 동일

당연한 것지만 static클래스에서만 static 변수 사용 가능 인스턴스 불가능 인스턴스 클래스는 당연 객체를 생성해야 사용가능한데
static은 객체 없이 사용가능하기 때문 모순이다.
단 final static int 와 같은 상수의 경우에는 static 제어자 가능 
이유: 객체 안에서만 쓰이는 것이 아니라면 static을 붙여야한다. 공통의 경우에는 static 사용한다는 특징을 잊지말자.

내부클래스에서의 제어자 접근성

내부클래스에서는 외부 클래스의 private 멤버에 접근 가능하지만 static은 역시나 불가능(iv)사용 못하기때문
메소드 내부클래스에서는 lv사용 불가능 이유는 메서드보다 내부 클래스가 더 오래 남아있을 경우 있어서 메서드종료시
지역변수가 소멸되면 지역변수를 내부클래스가 사용못한다. 단 변수인데 값이 안바뀌거나 상수인 경우는 사용가능.
